# 2026-04-24
<!-- DAILY_CHECKIN_2026-04-24_START -->

**《共识层规范》**

**以太坊权益证明（PoS）机制的“官方指导手册”，用 Markdown 格式编写的正式规范，定义了整个共识层从创世到现在所有升级的行为。**

**收集中心：它是所有以太坊核心共识规范的集合地（包含信标链、分叉选择、网络协议等）。**

**可执行：它会把自己“变”成能运行的 Python 代码，来验证自己写的对不对。**

**测试生成器：它能自动生成一套“标准答案”（参考测试向量），供各类客户端核对。**

**分叉策略**

**以太坊的共识层是通过一系列硬分叉（Hard Forks） 逐步演化的，规范的组织方式就体现了这种 “继承”关系。**

**分叉即“升级包”：每个新分叉的规范都基于上一版。**

**“继承式”编写：新版本只写新增或修改的部分，最后自动合并成完整规范。**

**指定生效时间：每个分叉都由一个“纪元编号（Epoch Number）”的数字精准控制。**

**信标链**

**CL 里最核心的链，是 PoS 机制的大脑。**

**它不处理DeFi、NFT等交易，只负责协调共识：管理验证者、维护注册表、随机选组长和评审团。**

`BeaconState`**（信标状态）就像总部的“活账本”，记录了验证者花名册、投票权重等实时动态；**

`BeaconBlock`**（信标区块） 记录了“选班长”、“投票结果”等元数据，而不是用户交易。**
