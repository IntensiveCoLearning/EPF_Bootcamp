# 2026-04-08

从 **SSZ 的固定区偏移量** 聊到 **Verkle 树的数学证明**，再到 **无状态性的去中心化博弈。**节点不再需要“爬”过整段交易，而是能像瞬移一样直接定位到第 500 笔交易的元数据，让多核 CPU 真正飞起来（并行计算）。Verkle Tree：无论树有多宽（256 叉），证明路径永远缩减为一个极小的数学证明（KZG 承诺）。它让“证据”的大小与“账本”的大小彻底脱钩。Statelessness：降低硬件门槛不是为了中心化，而是为了让全球几亿部手机都能随时通过数学公式**识别谎言**。这才是最高级的去中心化——**主权在于验证，而非占有。**用最少的流量、最快的速度、最硬的证据，达成最坚固的共识。

# 2026-04-06

今天学习了以太坊的来源以及那场令人振奋的密码朋克，在比特币网络上难以构建的应用程序，以太坊的出现就是为了解决这一问题。以太坊的协议架构，将CL/EL 分离，这样的系统架构，与Fabric 的 **Orderer（排序）** 与 **Peer（执行）** 分离的思想更是有异曲同工之妙。账户模型（Account-based）虽然在隐私保护上逊于 UTXO（如比特币），但它通过引入 **State（状态）** 的概念，让以太坊成为了一个巨大的“有限状态机”。在 Merkle 树中，你需要提供路径上所有的“邻居”节点；而在 Verkle 树中，证明体积显著缩小，这使得“**无状态客户端（Statelessness）**”成为可能——节点无需存储 2TB 的完整状态，仅凭一小段“见证数据（Witness）”即可验证交易。今天暂时学到这里，明天好好补上。
