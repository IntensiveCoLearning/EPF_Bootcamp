# 区块链学习打卡笔记：Execution Layer (EL) 与 EL Specs

📅 学习日期：今日  
📚 学习主题：Execution Layer（执行层）基础 + EL Specs（执行层核心规范）

* * *

## 一、Execution Layer（执行层）基础理解 🧠

Execution Layer（EL）是区块链（尤其是以太坊架构）中的**执行层**，主要负责处理用户交易和智能合约执行。

在当前以太坊模块化架构中，网络被分为两部分：

-   共识层（Consensus Layer，CL）
    
-   执行层（Execution Layer，EL）
    

两者分工明确：

共识层负责：

-   区块排序
    
-   出块规则
    
-   验证者管理
    
-   网络安全
    

执行层负责：

-   交易执行
    
-   状态更新
    
-   Gas 计算
    
-   智能合约运行
    
-   账户余额变化
    

一句话理解：  
👉 Execution Layer 是区块链真正“运行应用逻辑”的地方

* * *

## 二、Execution Layer 主要功能 ⚙️

### 1\. 交易执行

Execution Layer 接收用户提交的交易，例如：

-   转账
    
-   Token 交易
    
-   NFT 铸造
    
-   DeFi 操作
    

EL 负责：

-   验证交易合法性
    
-   执行交易逻辑
    
-   更新链上状态
    

* * *

### 2\. 智能合约执行 🤖

EL 内部包含虚拟机（如 EVM），用于执行智能合约代码。

执行过程：

1.  用户调用合约
    
2.  交易进入执行层
    
3.  虚拟机执行代码
    
4.  状态发生改变
    
5.  结果写入区块
    

* * *

### 3\. 状态管理 📊

执行层维护区块链状态，包括：

-   账户余额
    
-   合约存储数据
    
-   Token 持有情况
    
-   NFT 所有权
    

这些状态随着每笔交易不断更新。

* * *

### 4\. Gas 费用计算 ⛽

Execution Layer 负责：

-   计算每一步执行消耗
    
-   扣除用户 Gas
    
-   防止无限循环
    

Gas 的作用：

-   防止网络滥用
    
-   激励验证者
    
-   限制计算资源
    

* * *

## 三、Execution Layer 组件结构 🧩

Execution Layer 通常包括：

-   交易池（Transaction Pool）
    
-   执行引擎（Execution Engine）
    
-   状态数据库（State Database）
    
-   虚拟机（EVM）
    
-   RPC 接口
    

流程理解：

用户交易  
↓  
交易池  
↓  
执行引擎  
↓  
EVM 执行  
↓  
状态更新  
↓  
打包区块

* * *

## 四、EL Specs（执行层核心规范）理解 📘

EL Specs 是 Execution Layer 的技术规范，定义了：

-   交易格式
    
-   状态转换规则
    
-   Gas 计算方式
    
-   区块结构
    
-   执行结果标准
    

作用：  
👉 确保不同客户端执行结果一致

* * *

## 五、EL Specs 主要内容 🧠

### 1\. 状态转换函数（State Transition Function）

EL Specs 定义了：  
交易输入 → 状态变化 → 新状态

即：

旧状态 + 交易 = 新状态

所有执行层客户端必须遵循同一规则，否则链会分叉。

* * *

### 2\. 交易执行规则

EL Specs 规定：

-   Nonce 必须匹配
    
-   余额必须足够
    
-   Gas 限制检查
    
-   签名验证
    

只有符合规则的交易才会被执行。

* * *

### 3\. Gas 计算规范

EL Specs 明确：

-   每个操作消耗多少 Gas
    
-   存储写入成本
    
-   合约调用费用
    
-   内存扩展费用
    

这样所有节点计算一致。

* * *

### 4\. 区块执行结果

EL Specs 规定执行后必须生成：

-   State Root（状态根）
    
-   Receipt Root（收据根）
    
-   Logs Bloom
    
-   Gas Used
    

这些数据写入区块头。

* * *

## 六、Execution Layer 与钱包、Token、NFT 的关系 🔗

用户通过钱包发起交易  
↓  
交易进入 Execution Layer  
↓  
EL 执行智能合约  
↓  
Token / NFT 状态改变  
↓  
结果写入区块链

因此：

-   钱包 → 发起交易
    
-   Execution Layer → 执行逻辑
    
-   Token / NFT → 被执行对象
    

Execution Layer 是资产变化真正发生的地方。

* * *

## 七、今日学习核心收获 🎯

通过今天学习，我理解了：

1.  Execution Layer 是区块链的执行引擎
    
2.  所有交易和智能合约都在 EL 中运行
    
3.  EL 负责状态更新与 Gas 计算
    
4.  EL Specs 是执行层统一规则
    
5.  不同客户端必须遵循 EL Specs
    
6.  钱包操作最终都会进入 Execution Layer
    
7.  Token 和 NFT 的变化由 EL 执行产生
    

* * *

共识层（CL） → 决定谁出块  
执行层（EL） → 执行交易

应用资产：

Token / NFT → 链上资产

用户入口：

钱包 → 发起交易

完整流程：

钱包 → 交易 → Execution Layer → 智能合约 → Token/NFT变化 → 写入区块
