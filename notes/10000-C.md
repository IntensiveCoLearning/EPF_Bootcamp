## QR 协议的 Real View 与 Simulator 完整证明过程


### 一、QR 协议回顾

**声明**：$y$ 是模 $N$ 的二次剩余，即 $\exists x \text{ s.t. } y = x^2 \bmod N$

证明者 P 知道秘密 $x$，一轮交互如下：

| 步骤 | P（证明者） | V（验证者） |
|---|---|---|
| ① | 随机选 $r \in \mathbb{Z}_N^*$，计算 $s = r^2 \bmod N$，发送 $s$ | |
| ② | | 抛硬币 $b \xleftarrow{\$} \{0,1\}$，发送 $b$ |
| ③ | 若 $b=1$：发送 $z = r$；若 $b=0$：发送 $z = r \cdot x \bmod N$ | |
| ④ | | 验证 $z^2 \stackrel{?}{=} s \cdot y^{1-b} \bmod N$ |

**验证等式检查**：
- $b=1$ 时：检查 $z^2 = s \cdot y^0 = s = r^2$ ✅（因为 $z=r$）
- $b=0$ 时：检查 $z^2 = s \cdot y^1 = r^2 \cdot x^2 = (rx)^2$ ✅（因为 $z=rx$）

---

### 二、Real View 的分布

一轮的真实视图为三元组 $(s, b, z)$，其生成过程：

```
r ← Z_N*  (均匀随机)
s = r² mod N
b ← {0,1}  (均匀随机)
if b=1: z = r
if b=0: z = r·x mod N
输出 (s, b, z)
```

我们需要分析 $(s, b, z)$ 的**联合概率分布**。

**关键观察**：$x \in \mathbb{Z}_N^*$（因为 $y = x^2 \bmod N$，$y \neq 0$），所以"乘以 $x$"是 $\mathbb{Z}_N^*$ 上的一个**置换**（bijection）。

因此：
- **$b=1$ 时**：$z = r$ 均匀分布于 $\mathbb{Z}_N^*$，$s = z^2$
- **$b=0$ 时**：$z = r \cdot x$，由于乘 $x$ 是置换，$z$ 仍然均匀分布于 $\mathbb{Z}_N^*$，$s = (z/x)^2 = z^2 \cdot y^{-1} \bmod N$

所以 Real 分布可以写成：

$$\text{Real} = \begin{cases} (z^2,\ 1,\ z) & \text{w.p. } 1/2,\quad z \xleftarrow{\$} \mathbb{Z}_N^* \\ (z^2 \cdot y^{-1},\ 0,\ z) & \text{w.p. } 1/2,\quad z \xleftarrow{\$} \mathbb{Z}_N^* \end{cases}$$

---

### 三、Honest Verifier 的 Simulator（课件第 40 页）

Simulator **不知道 $x$**，构造如下：

```
1. 随机选 b ← {0,1}
2. 随机选 z ← Z_N*
3. 计算 s = z² / y^{1-b} mod N
   即：若 b=1，s = z²
       若 b=0，s = z² · y^{-1}
4. 输出 (s, b, z)
```

**核心技巧**：Sim 是"倒着构造"的——先选 $z$ 和 $b$，再反推 $s$，使得验证等式自然成立。

Simulated 分布：

$$\text{Simulated} = \begin{cases} (z^2,\ 1,\ z) & \text{w.p. } 1/2,\quad z \xleftarrow{\$} \mathbb{Z}_N^* \\ (z^2 \cdot y^{-1},\ 0,\ z) & \text{w.p. } 1/2,\quad z \xleftarrow{\$} \mathbb{Z}_N^* \end{cases}$$

---

### 四、Perfect ZK 证明（Honest Verifier）

**结论**：$\text{Real} \equiv \text{Simulated}$（完全相同的分布，不是"近似"，是"等于"）

**证明**：逐行对比上面的两个分布表达式——**完全一样**。✅

这就是为什么叫 **Perfect** Zero Knowledge：模拟出来的和真实的不是"看起来像"，而是"概率分布完全相同"，任何算法（不限于 PPT）都无法区分。

---

### 五、恶意验证者 V* 的 Simulator（课件第 41 页）

**难点**：恶意 V* 不一定均匀随机选 $b$。V* 是一个 PPT 算法，看到 $s$ 后可能用某种策略输出 $b'$。

**Simulator 构造**：

```
输入：(N, y)
循环：
  1. 随机选 b ← {0,1}
  2. 随机选 z ← Z_N*
  3. 计算 s = z² / y^{1-b}
  4. 运行 V*((N,y), s)，得到 b'
  5. 如果 b' = b，输出 (s, b, z)，停止
  6. 否则，回到步骤 1，重试
```

**为什么期望只需 2 次迭代？**

关键在于证明 $\Pr[b' = b] = 1/2$。分析如下：

**Lemma**：无论 Sim 选了 $b=0$ 还是 $b=1$，产生的 $s$ 的分布**完全相同**——都是 $\text{QR}_N$（模 $N$ 的二次剩余集）上的均匀分布。

证明：
- $b=1$ 时：$s = z^2$，$z$ 均匀于 $\mathbb{Z}_N^*$，所以 $s$ 均匀于 $\text{QR}_N$ ✅
- $b=0$ 时：$s = z^2 \cdot y^{-1}$。因为 $y \in \text{QR}_N$，所以 $y^{-1} \in \text{QR}_N$。$\text{QR}_N$ 是一个群（子群），所以 $\text{QR}_N \cdot y^{-1} = \text{QR}_N$。因此 $s$ 也均匀于 $\text{QR}_N$ ✅

由于 V* 看到的 $s$ 的分布**与 $b$ 无关**，V* 对 $b$ 的"猜测" $b'$ 等于 $b$ 的概率恰好是 $1/2$。

因此迭代次数服从几何分布，期望为 $1/(1/2) = 2$ 次。✅

**为什么输出仍然分布正确？**

条件在 $b' = b$ 上，输出的 $(s, b, z)$ 的分布与 honest verifier 情况完全相同（因为只是多了一个"过滤条件"，而过滤条件对两个分支是对称的）。

所以即使是恶意 V*，Simulator 也能产生与 Real **完全相同**的分布——**Perfect ZK**。✅

---

### 六、总结对比图

```
Real View 的生成方向：     r → s=r² → 收到b → z → 输出(s,b,z)
                          (正向，需要知道x)

Simulated View 的生成方向：b,z → s=z²/y^{1-b} → 输出(s,b,z)
                          (反向，不需要知道x)

两者的分布：完全相同 ≡
```

这就是 QR 协议是 Perfect ZK 的完整证明。整个论证的核心洞察是：**s 在两种情况下都是 QR_N 上的均匀分布**，这保证了 Simulator 可以"倒着构造"而不被任何验证者察觉。

## Fiat-shamir使用哈希函数生成非交互式零知识证明

1. 原交互式零知识证明中，验证者发出的随机挑战是安全性的关键——证明者必须先承诺（发送第一步消息），然后才能看到挑战，无法根据挑战回头修改承诺

2. Fiat-Shamir 用哈希函数 $b = H(x, a_1)$ 替代验证者的随机挑战。哈希函数的不可预测性保证证明者无法先选好想要的 $b$ 再反推 $a_1$；其抗原像性保证证明者无法找到特定 $a_1$ 来操控 $b$ 的值。证明者必须先固定 $a_1$，被动接受 $H$ 决定的挑战——这与交互式中的"先承诺后看到挑战"结构等价

3. 对于验证者，$b = H(x, a_1)$ 是确定性的、可独立复现的。验证者不需要相信证明者提供的挑战值，而是自己计算 $b$ 然后验证——证明者无法提交一个与 $H(x, a_1)$ 不匹配的响应而通过验证
